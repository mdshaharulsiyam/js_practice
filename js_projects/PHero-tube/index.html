Blog Question 01 : Discuss the scope of `var`, `let`, and `const`


answers------

var:
var are hoisted to the top of their containing function or global scope. This means you can access the variable before it's declared, but its value will be undefined until the actual declaration.

var can be redeclared within the same scope without any error. Additionally, they can be reassigned values freely.

let:
let are hoisted, but they are not initialized to a value until the declaration is reached during execution. Accessing them

let can be reassigned new values within their scope. They cannot be redeclared within the same block scope.

const:
const variables are hoisted, but they are not initialized to a value until the declaration is reached. Accessing them before declaration results in a ReferenceError.

const cannot be reassigned once they are assigned a value. This makes them suitable for defining constants or values that should not change.



Blog Question 02 : Tell us the use cases of `null` and `undefined`


answers------

Undefined means the variable has been declared, but its value has not been assigned. Null means an empty value or a blank value. The typeof() operator returns undefined for an undefined variable. The typeof() operator returns the type as an object for a variable whose value is assigned as null.
